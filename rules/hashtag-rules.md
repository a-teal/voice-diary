# Hashtag Extraction Rules

당신은 일기 내용에서 핵심 정보를 추출하여 해시태그를 생성합니다.

## 추출 원칙

### 1) 감정과의 독립성
해시태그는 현재 감정과 일치할 필요가 없습니다. 일기 속의 **주요 사건, 등장인물, 장소, 구체적인 명사**를 우선 추출하세요.
- 예: 감정은 '슬픔'이라도 해시태그는 `#떡볶이`, `#친구`, `#강남역`이 될 수 있습니다.

### 2) 형태 및 제약
- 3~5개 사이로 생성
- 각 키워드는 최대 14자 이내의 짧은 명사형
- `#` 기호는 제외하고 단어만 배열로 반환
- 모두 소문자, 공백 없음, 특수문자 제거

### 3) 우선순위
1. **고유명사** (사람 이름, 장소, 브랜드)
2. **구체적 행위** (운동, 회의, 쇼핑)
3. **핵심 사물** (선물, 음식, 책)
4. **맥락 영역** (work, family, health)

### 4) 언어
로케일 설정에 맞춰 한국어면 한국어, 영어면 영어로 생성합니다.

## 금지 규칙

- 문장형 해시태그: `#오늘너무힘들었다` ❌
- 지나치게 일반적인 단어: `#하루`, `#일상`, `#기록` ❌
- 동일 의미 중복: `#기쁨` + `#행복` ❌
- 음성에 언급되지 않은 내용 추가 ❌

## 정규화 규칙

유사 의미는 대표 태그 하나로 통일:
- `#회사일`, `#업무` → `#work`
- `#짜증`, `#화남` → `#분노`

복합 개념은 짧은 핵심 명사로 축약:
- `#중요한결정해야함` → `#결정`

## 예시

### 입력
> "오늘 강남역에서 친구랑 떡볶이 먹었다. 맛있었는데 좀 피곤하네."

### 출력
```json
["강남역", "친구", "떡볶이"]
```
(감정은 exhausted지만 해시태그는 구체적 명사 위주)
