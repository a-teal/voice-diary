음성으로 빠르게 기록하고 AI가 구조화해주는 접근이 "귀찮음"이라는 핵심 문제를 정확히 해결하는 것 같습니다.

이제 구체적으로 개발 계획을 세워봅시다.

## 기술 스택 제안

**프론트엔드**
- React/Next.js (웹) 또는 React Native (모바일 우선이라면)
- 음성 녹음: Web Speech API 또는 react-native-voice
- 차트: Recharts 또는 Chart.js

**백엔드**
- Node.js/Express 또는 Python/FastAPI
- 음성→텍스트: OpenAI Whisper API 또는 Google Speech-to-Text
- AI 분석: Claude API 또는 GPT API (키워드 추출, 감정 분석)
- DB: PostgreSQL (구조화된 데이터) + Vector DB (향후 검색 고도화)

**호스팅**
- Vercel/Netlify (프론트) + Railway/Render (백엔드)
- Supabase (DB + Auth 통합 솔루션)

## MVP 개발 우선순위

**Phase 1 (2주): 핵심 기능**
1. 음성 녹음 → 텍스트 변환
2. AI 분석 (키워드 3-5개, 감정 이모지 1개)
3. 일간 뷰 (텍스트 + 분석 결과)
4. 간단한 저장/불러오기

**Phase 2 (1-2주): 시각화**
5. 월간 캘린더 뷰 (이모지 표시)
6. 주간 키워드 클라우드
7. 색상 그래프 (감정 흐름)

**Phase 3 (1주): 연간 통계**
8. 연간 뷰 구현
9. 네비게이션 완성

## 몇 가지 결정이 필요한 사항

**1. 플랫폼 우선순위**
- PWA 웹앱으로 시작 → 모바일처럼 사용 가능 + 빠른 개발

**2. 음성 녹음 방식**
- 실시간 스트리밍 변환 (즉각 피드백)

**3. AI 프롬프트 설계**
감정 이모지는 AI가 자유롭게 선택!

**4. 데이터 프라이버시**
- 로그인 필수는 아니고, 로컬 저장!

**5. 수익 모델** (나중이라도)
- 무료 + 프리미엄 (연간 통계, 고급 분석)
