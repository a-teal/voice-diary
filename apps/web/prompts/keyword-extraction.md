# Keyword Extraction Rules

## 목적
일기 텍스트에서 의미 있는 키워드(해시태그)를 추출하여 검색 및 통계에 활용

## 기본 규칙

### 1) 개수
- 최소: 3개
- 최대: 5개
- 항상 3개 이상 반환 (부족하면 fallback 사용)

### 2) 형태
- 짧은 명사형 (1~3어절)
- 최대 14자
- 중복 불가
- 문장 형태 금지

### 3) 언어
- locale="ko": 한국어 키워드
- locale="en": 영어 키워드

## 추출 우선순위

1. **핵심 주제** (무엇에 대한 이야기인가)
   - 사람: 친구, 가족, 동료, 엄마, 아빠
   - 장소: 회사, 학교, 집, 카페
   - 활동: 운동, 공부, 일, 여행

2. **감정 원인** (왜 그런 감정인가)
   - 스트레스, 성취감, 피로, 기대

3. **구체적 명사** (언급된 것들)
   - 음식, 날씨, 건강, 취미

## 제외 대상

- 조사/어미: 은, 는, 이, 가, 를, 에서
- 일반적 동사: 하다, 되다, 있다, 없다
- 시간 부사: 오늘, 내일, 어제 (단, 특별한 의미가 있으면 포함)
- 감탄사: 아, 음, 와

## Fallback 키워드

키워드를 추출하지 못한 경우:
```
['일상', '기록', '오늘']
```

## 예시

### 입력
"오늘 회사에서 프로젝트 마감 때문에 야근했다. 피곤하지만 뿌듯하다."

### 출력
```json
{
  "keywords": ["회사", "프로젝트", "야근", "마감"]
}
```

### 입력
"친구랑 맛있는 파스타 먹었다. 오랜만에 만나서 좋았다."

### 출력
```json
{
  "keywords": ["친구", "파스타", "만남"]
}
```
